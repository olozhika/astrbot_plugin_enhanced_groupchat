# EnhancedGroupChat (打破前缀/唤醒词限制的群聊强化深度互动插件)

根据人类行为设计的AstrBot群聊交互强化插件。避免传统聊天机器人“必须被 @ 或输入特定指令/前缀才回复”的单向死板交互模式，引入了**非触发闲聊、多群独立概率、连续窥屏追击、智能上下文消息裁剪**等高级行为逻辑，让大模型在群聊中呈现出更加自然、生动、且兼顾成本控制的人性化互动。

---


## 🔥 核心特性

### 1. 深度多群概率闲聊设置 (`n`)
* 通常群聊中，用户不使用前缀或不 `@` 机器人时，机器人都充当隐形人。本插件允许通过配置对普通聊天的随机行为进行控制。
* **分群独立概率控制**：全新支持格式为 `[[群号, 几率倒数 n], [群号, 几率倒数 n]]` 的配置表，可以让不同群聊拥有不同的活跃度。如果在列表中没有找到当前群聊，则回退使用默认值（10）。
* **兼容性设计**：依然向下兼容直接输入单个整数 `n`。若设为 1，则每一条普通群聊消息都会触发回复；若设为 10，则每次触发率为 1/10 (10%)。

### 2. 连续窥屏 (Peeping) 与追击连击机制 (`M` / `N`)
* **像真人一样插话**：一旦机器人通过概率触发或 `@AI` / `\` 等普通手段在群里进行了一次发言，它就会自动切入**"连续窥屏关注阶段"**。
* **时效性关注 (`M`)**：在此期间内的任何后续发言，都在机器人的时刻关注下。只要群友在 `M` 分钟（窥屏超时时长）内继续发言，机器人就有可能无须再次概率判定，直接自然插话追击回复，模拟沉浸式多人对话。
* **安全防刷阀门 (`N`)**：为防止机器人在高频刷屏群中由于连续回复导致"刷屏死循环"，在到达最大连续窥屏时长 `N`（分钟）上限后，机器人会自动强制退出关注模式，退归至普通 1/n 概率模式，进行优雅退火。

### 3. 长未读消息智能修剪与安全注入 (`L` / `keep_keywords`)
* **费用克星**：群友聊得特别嗨，半天没触发机器人，等下一次触发时，可能有上百条冗余未读消息一次性灌入大模型上下文。
* **核心内容保留策略**：在触发回复的一瞬间，若大模型上一次回复后的未读消息过多，自动执行极简剪裁逻辑，仅保留：
  1. **最后的 L 条** 最新消息。
  2. **紧随上一次回复后的 5 条** 起始消息（保留对话承上启下的开端）。
  3. **含特定自定义保留关键词的消息**（如群友提及机器人的名字、称呼、或核心设定等，确保不会遗漏与机器人强相关的关键上下文）。
* **上下文长度平滑化 (提示词重置)**：本插件会在裁剪的部分自动插入一段格式化的系统过渡提示 `"<system_reminder>系统提示：由于未读消息过长，此处已自动略过 {gap} 条普通聊天记录，以避免上下文过长。</system_reminder>"`，使 LLM 能完美感知到群聊上下文的折叠和时空跨越，极大地节省 API 费用同时不对生成效果造成负面影响！

### 4. 统一昵称 (`uni_nickname`) 深度联动
* 完美打破第三方平台（如 QQ 原始 ID、微信混淆 ID）的冰冷面具，能够自动检测并读取 `astrbot_plugin_uni_nickname_config.json` 映射配置。
* 当机器人在大模型提示词中拼装发言者聊天记录时，会自动提取统一昵称插件配置中关联的自定义群友好称（如"张三"、"群主爸爸"），使机器人在理解和生成回答时富有温度、精准认人，完美契合熟人社交与长期记忆插件的联动！

### 5. 群白名单运营控制 (`white_groups`)
* 避免将机器人大范围投放到无关的群聊中，通过支持用逗号、空格分隔的专属白名单群号，确保插件的精准作用和隐私安全。

### 6. 模拟人类打字延迟发送 (`typing_speed_coefficient`)
* AI 写完回复后，根据回复字数模拟人类打字速度延迟发送。回复内容越长，发送越慢，让机器人的发言节奏更接近真人。
* 计算公式：`延迟(秒) = 回复字数 / 3.0 × 系数`（3.0 为每分钟约 180 字的基准打字速度）。
* 系数可在配置中自由调节：设为 `0` 关闭该功能；`< 1` 打字更快；`> 1` 打字更慢。默认为 `1.0`。

## ⚙️ 插件配置项说明

本插件支持在 AstrBot Web 后台插件市场 / 仪表盘中，可视化修改相关的 JSON 模式化设置信息：

| 键名 | 描述 | 类型 | 提示 & 范例说明 | 默认值 |
| :--- | :--- | :--- | :--- | :--- |
| **`n`** | 群聊无触发闲聊回复几率倒数列表 | `list` / `int` | `[[群号, 几率倒数 n], [群号, 几率倒数 n]]`。若列表中没有匹配当前群，则采用 10。<br>例：`[["123456", 10], ["789012", 5]]` | `[["123456", 10], ["789012", 5]]` |
| **`M`** | 窥屏超时时长 M (分钟) | `float` | AI 回复后，在此时间内如果没有该群聊的新消息，则 AI 退出连续窥屏状态，回退到 1/n 的几率回复。 | `3.0` (3分钟) |
| **`N`** | 最大连续窥屏时长 N (分钟) | `float` | AI 连续进行窥屏并追踪回复的最大时间上限。一旦超过，AI 自动退出窥屏。防止群聊高频消息导致刷屏。 | `5.0` (5分钟) |
| **`L`** | 未读消息最长注入 L (条) | `int` | 当触发回复时，限制积攒未读消息过多时的上限。保留最近的 L 条未读。 | `15` |
| **`keep_keywords`** | 未读消息保留关键词列表 | `string` | 在截断被裁剪的上下文时，凡是包含这些关键词的句子，无论多么靠前都一律保留。例：`"AI, 机器人, 小助手"` | `""` |
| **`white_groups`** | 启用群白名单 | `string` | 英文逗号或空格分割的群号。为空则代表面向全平台的全部群聊开启本插件功能。 | `""` |
| **`typing_speed_coefficient`** | 模拟打字速度系数 | `float` | AI 回复后根据字数延迟发送。系数越大越慢，`0` 关闭。公式：字数 / 3 × 系数。 | `1.0` |

## 🛠️ 安装与部署

1. **一键安装**：直接在 AstrBot 的插件市场或商店搜寻 `enhanced_groupchat`，点击安装。
2. **源码放置**：克隆或下载本插件目录，将其放置在你的 AstrBot 插件工作区（通常是 `/addons/plugins/` 目录）下，将目录名保持为 `enhanced_groupchat`。
3. **重启应用**：修改后重启 AstrBot 服务，控制台看到 `[EnhancedGroupChat] 群聊强化插件已成功加载！` 字样即成功。
4. **进阶推荐**：强烈建议安装配套的 **[统一昵称](https://github.com/Hakuin123/astrbot_plugin_uni_nickname)** 模块，获得极佳的一体化人设群聊体验。

## 🙋 常见问题答疑 (FAQ)

### Q1: 普通发言概率很低，怎么快速测试它？
可以在配置项里将对应群组的 `n` 改成 `1` 或 `2`。这样只要有任意群友在群里说哪怕一个表情或一个字，都会高频触发回复机制，可以快捷验证是否成功生效。

### Q2: 自定义保留词 `keep_keywords` 应该填什么？
建议填写机器人在群里的**名字、触发指令语、以及它的人设名称**。比如你通过系统 Prompt 设置机器人的名字叫 `"可莉"` 或 `"初音"`，则可以在 `keep_keywords` 里填写 `"可莉, 太阳"` 等。如此即便聊天间隔过了几十页，只要有人讨论到和它相关的内容，在上下文注入时这几句话和语境就不会被略去。

### Q3: 为什么有的群不触发？
请核对你的 `white_groups` 配置。如果你在此项填写了具体的群号，那么不在白名单中的群组消息将被自动过滤而无法触发任何智能行为。此外，请检查 `AstrBot` 官方主程序对该群的授权。

### Q4: 模拟打字延迟 (`typing_speed_coefficient`) 会影响消息记录吗？
不会。在打字延迟期间，新接收到的群聊消息会暂存在内存缓冲区中，延迟结束后统一刷新到聊天记录数据库中，确保不会丢失任何消息。同时插件内部自动刷新了超时熔断计时器，即使延迟时间较长也不会误触发安全保护。
### Q2: 自定义保留词 `keep_keywords` 应该填什么？
建议填写机器人在群里的**名字、触发指令语、以及它的人设名称**。比如你的AI名字叫 `“Lanya”`，则可以在 `keep_keywords` 里填写 `“Lanya, lanya, 小AI”` 等。如此即便聊天间隔过了几十页，只要有人讨论到和它相关的内容，在上下文注入时这几句话和语境就不会被略去。

### Q3: 为什么有的群不触发？
请核对你的 `white_groups` 配置。如果你在此项填写了具体的群号，那么不在白名单中的群组消息将被自动过滤而无法触发任何智能行为。此外，请检查 `AstrBot` 官方主程序对该群的授权。
